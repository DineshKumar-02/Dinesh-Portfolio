Application Architecture

ThinkHub follows a MERN Stack architecture structure. A client application designed in React communicates with an Express and Node server which tracks collection indices in a MongoDB Atlas database using Mongoose models.

[React.js Frontend] ──(REST API Calls)──> [Node/Express Server] │ [Mongoose ODM] │ [React.js Frontend] <──(JSON Responses)─── [MongoDB Atlas Database]

Key Features

  • User Signup: Verification logic ensuring fields (Name, Email, Age >= 13) match rules before database commits.
  • Category Filtering: Dashboard structured into 14 distinct channels to ensure straightforward post searches.
  • CRUD Router: Backend routes supporting safe document insertions, collection queries, and document removals.
  • Cloud Deployment: Deployed static React files to Netlify and configured Render scripts for API server endpoints.

Challenges and Resolutions

  • Database CORS alerts: Handled cross-domain security bans by configuring appropriate middleware filters inside express setups.
  • Reload routing drops: Avoided 404 navigation drops on Netlify manual page reloads by establishing correct rewrite mappings in local _redirects configs.
  • API URL Consistency: Used environment configurations to switch clean endpoint endpoints during development and deployment transitions.
